네이밍 규칙:

http://www.w3.org/TR/html401/types.html#type-name

ID and NAME tokens must begin with a letter ([A-Za-z]) and may be followed by any number of letters, digits ([0-9]), hyphens ("-"), underscores ("_"), colons (":"), and periods (".").

반드시 알파벳으로 시작하고, 그 다음부터는 숫자, 하이픈, 언더스코어, 콜론, 피리어드 등이 올 수 있다.

html5에서는 조금 다르게 설명되어 있다.

http://www.w3.org/TR/2011/WD-html5-20110525/elements.html#the-id-attribute

The id attribute specifies its element's unique identifier (ID). The value must be unique amongst all the IDs in the element's home subtree and must contain at least one character. The value must not contain any space characters.

1) 문서에서 유일한 값이어야 하고, 2) 하나 이상의 문자가 있어야 하고, 3) 공백이 들어가면 안된다.

HTML 4에서 정의된 것보다 좀 유연하다.

또 다른 규칙:

http://www.w3.org/TR/html401/types.html#case-sensitive

The value is case-sensitive (i.e., user agents interpret "a" and "A" differently).

대소문자 구분한다.
