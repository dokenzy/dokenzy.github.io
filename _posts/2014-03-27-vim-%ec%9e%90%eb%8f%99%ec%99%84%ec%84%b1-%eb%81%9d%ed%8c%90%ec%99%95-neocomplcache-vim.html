며칠 전부터 갑자기 MacVim에서 코드 자동완성이 안되었다.  그 때 했던 거라곤 한글입력기 테스트 하느라 하늘입력기, 바람입력기, 구름입력기 등을 테스트하다 MacUIM에서 Shift+Space로 한영전환이 안되서 삽질했던 때다.

또 하나는 누군가가 Vim에서 HTML코드 내 Javascript 코드의 자동들여쓰기가 안된다는 글을 올려서 관련 플러그인을 설치했었다.

둘 다 해결했지만, 이 쯤부터 MacVim에서 자동완성이 안되기 시작했다. 내가 쓰던 자동완성 기능은 PyDiction처럼  Python 키워드 등을 미리 저장해 두고 꺼내오는 게 아니라, 편집중인 코드의 버퍼에서 자동으로 가져오는 기능이 있었다. PyDiction과는 비교할 수 없이 편했다.

오늘 삽질 끝에 딱 내가 찾던 자동완성 플러그인을 찾았다. 편리함으로 보자면 기존 것이 더 편했지만, <Tab>키의 작동방식은 새로 찾은 것이 더 낫다. 몇 글자 입력 후 <Tab>키를 누르면 자동완성리스트가 뜬다. <Tab>키를 누를 때마다 다음 항목으로 이동하고, 맨 마지막 항목에서 다시 <Tab>키를 누르면, 처음 입력한 상태로 돌아간다.

이 플러그인의 이름은 neocomplcache

설치는 다른 Vim 플러그인처럼 Pathogen을 이용하면 편리하게 할  수 있다.

cd ~/.vim/bundle
git clone https://github.com/Shougo/neocomplcache.vim.git

이제 .vimrc 파일에 설정을 한다. 나는 요 정도만 해주었다.

let g:pymode_breakpoint = 1
let g:pymode_breakpoint_bind = '<leader>b'
let g:pymode_breakpoint_cmd = ''
let g:pymode_rope_completion = 1
let g:pymode_rope_complete_on_dot = 1
let g:pymode_folding = 0
let g:pymode_rope_guess_project = 0
let g:pymode_virtualenv = 1
let g:pymode_run = 1
let g:pymode_indent = 1
let g:pymode_lint = 1
let g:pymode_lint_unmodified = 1
let g:pymode_lint_checker = "pyflakes,pep8,mccabe"
let g:pymode_lint_mccabe_complexity = 8
let g:pymode_lint_ignore = "E501"
let g:pymode_lint_cwindow = 1
let g:pymode_lint_message = 1

참고로 Vim 버전이 7.3.885 이상이고, Lua를 사용할 수 있다면 neocomplete.vim를 쓰는 게 더 빠르다고 한다. newcomplete.vim을 쓸 수 있는지 확인하려면 Vim에서 :echo has("lua") 를 실행해서 1 이 나오는지 테스트한다.
