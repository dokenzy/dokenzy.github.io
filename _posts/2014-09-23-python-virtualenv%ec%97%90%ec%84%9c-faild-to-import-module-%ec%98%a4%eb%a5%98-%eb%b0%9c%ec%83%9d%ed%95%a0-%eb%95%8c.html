맥 OS X(Mavericks)에서 virtualenv로 새 환경을 만들려는데 아래같은 오류가 발생한다.

Running virtualenv with interpreter .pythonz/pythons/CPython-3.4.1/bin/python
Using base prefix '/Users/username/.pythonz/pythons/CPython-3.4.1'
New python executable in venvs/proj/bin/python
Failed to import the site module
Traceback (most recent call last):
  File "/Users/username/venvs/proj/lib/python3.4/site.py", line 67, in <module>
    import os
  File "/Users/username/venvs/proj/lib/python3.4/os.py", line 614, in <module>
    from _collections_abc import MutableMapping
ImportError: No module named '_collections_abc'
ERROR: The executable venvs/proj/bin/python is not functioning
ERROR: It thinks sys.prefix is '/Users/username' (should be '/Users/username/venvs/proj')
ERROR: virtualenv is not compatible with this system or executable

virtualenv(+pythonz)로 새로운 환경을 만들려는데 이런 오류가 발생한다.

해결방법: virtualenv를 업데이트한다.

sudo pip install --upgrade virtualenv

관련 링크: error with virtualenv and python 3.4a1 #463

근데 이상하다. 최근에 회사에서 쓰는 맥북에서는 이런 문제가 없었는데...
(둘 다 CPython 3.4.1)
