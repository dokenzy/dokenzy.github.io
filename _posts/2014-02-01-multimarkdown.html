MultiMarkdown이란?

MultiMarkdown(MMD)는 미니멀한 마크업 텍스트를 다른 포맷의 문서로 변환해주는 도구다. HTML, PDF(LaTeX을 이용해서), OPML 그리고 OpenDocument도 된다. 특히 Flat OPML은 RTF, 마이크로소프트 Word, 그리고 가상의 다른 워드프로세서로 변환할 수 있는 포맷이다.

MMD는 John Gruber가 만든 Markdown의 수퍼셋이다. 위에서 언급한 다양한 출력 포맷을 지원하기 위해 표, 주석, 인용같은 여러 기능을 추가했다(Markdown은 HTML으로만 변환할 수 있다). 또, 다국어를 위한 “똑똑한” 타이포그래피 기능도 있다(예를 들면 왼쪽-오른쪽 따옴표같은 것).

MultiMarkdown은 원래 Markdown.pl을 수정한 Perl 스크립트에서 시작했다.

MultiMarkdown v3(peg-multimarkdown이라고도 알려진)는 John MacFarlane의 peg-markdown을 기반으로 한다. 이것은 파싱표현식문법(PEG)을 사용하는데, 대부분의 운영체제에서 컴파일할 수 있도록 C로 작성됐다. Daniel Jalkut 덕분에 MMD v3는 외부 라이브러리 필요없이 빌드할 수 있도록 만들어졌다.

MultiMarkdown v4는 기본적으로 v3를 완전히 다시 작성했다. (Multi)Markdown을 파싱하기 위해 기본적인 PEG은 같은 것을 사용한다. 하지만 거의 다시 만들어졌다.

  • 코드를 유지보수하기 쉽게 설계했다. 더 논리적인 구조에 맞춰 파일을 분리했다.
  • (내가 알고 있는)메모리 누수 문제를 모두 수정했다.
  • 파서를 만들 때 [peg/leg] 대신 [greg]을 사용한다. 이렇게 하면 파서가 thread-safe된다.
  • 몇 가지 개선을 하기 위해 [test suite]가 바꼈다. MMD는 기본 Markdown 테스트중에 하나에서 실패할 것이다(자세한 건 peg-markdown을 참조).
  • 커맨드 라인 옵션이 조금 바꼈다.

MultiMarkdown이 무엇인지에 대한 다른 설명은 PDF 슬라이드에서 확인할 수 있다. MultiMarkdown을 사용하는 방법과 데모를 볼 수 있다.

왜 MultiMarkdown을 사용해야 하나?

MultiMarkdown으로 작성하면 문서의 내용과 구조를 모양으로부터 분리할 수 있다. 장절제목이나 문단 간격같은 스타일을 만드는 데 신경쓰지 않고 글쓰기에만 집중할 수 있다. 그리고 좀 더 생각해보면, 하나의 플레인 텍스트 문서는 여러 출력 포맷으로 변환할 수 있다. 일일이 문서 전체나 포맷을 다시 쓸 필요가 없다. 더 좋은 점은, 좋은 HTML문서나 LaTeX 명령을 만들기 위해 “컴퓨터 언어”로 쓸 필요가 없다. 여러분은 글만 쓰면 된다. 나머지는 MultiMarkdown이 해준다.

예를 들면, 이렇게 쓰는 대신

<p>In order to create valid 
<a href="http://en.wikipedia.org/wiki/HTML">HTML</a>, you 
need properly coded syntax that can be cumbersome for 
&#8220;non-programmers&#8221; to write. Sometimes, you
just want to easily make certain words <strong>bold
</strong>, and certain words <em>italicized</em> without
having to remember the syntax. Additionally, for example,
creating lists:</p>

<ul>
<li>should be easy</li>
<li>should not involve programming</li>
</ul>

그냥 이렇게 쓰면 된다.

In order to create valid [HTML], you need properly
coded syntax that can be cumbersome for 
"non-programmers" to write. Sometimes, you just want
to easily make certain words **bold**, and certain 
words *italicized* without having to remember the 
syntax. Additionally, for example, creating lists:

* should be easy
* should not involve programming

[HTML]: http://en.wikipedia.org/wiki/HTML

게다가, 아무 운영체제나 텍스트 편집기에서 MultiMarkdown 문서를 작성할 수 있다. 어떤 운영체제에서 작성해도 호환되기 때문에 출력이 같다. 플레인 텍스트 포맷이기 때문에, 컴퓨터, 운영체제, 응용프로그램들끼리 아무리 왔다갔다 해도 안전하다. 처음 문서를 작성한 소프트웨서가 한참 전에 사라지더라도 항상 문서를 열고 편집할 수 있다.

이런 기능 때문에 많은 사람들이 자신의 책, 논문, 기타 여러 문서를 작성하는 데 멀티마크다운을 사용하기 시작했다.

MultiMarkdown을 사용할 이유가 많이 있지만, 여기서 전부 다루지는 않겠다.
