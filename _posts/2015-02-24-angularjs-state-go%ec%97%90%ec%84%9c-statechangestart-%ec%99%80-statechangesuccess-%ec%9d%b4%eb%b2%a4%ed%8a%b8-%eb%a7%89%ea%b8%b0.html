AngularJS에서 ui-router를 쓸 때, $state의 변화를 줄 때(?) .go() 함수를 쓰는데, 이걸 쓰면 기본적으로 $stateChangeStart 와 $stateChangeSuccess 이벤트가 발생한다. 여러 디렉티브를 함께 쓰다보면 이 이벤트 때문에 원하지 않는 기능이 자동으로 수행된다.

go()는 내부적으로 $state.transitionTo()를 호출하는데, 이 때 다음 옵션을 기본값으로 사용한다.

{ location: true, inherit: true, relative: $state.$current, notify: true }

따라서 go()를 호출할 때 옵션을 설정해 주면 기본적으로 발생하는 $stateChangeStart 와 $stateChangeSuccess 이벤트를 막을 수 있다. 이 이벤트를 막는 옵션은 notify를 false로 주면 된다.

즉, 다음과 같은 형태로 사용하면 된다.

$state.go('detail', {id: myId}, {notify: false});

그럼 URL은 바뀌지만 $stateChangeStart 와 $stateChangeSuccess 이벤트는 발생하지 않는다.
