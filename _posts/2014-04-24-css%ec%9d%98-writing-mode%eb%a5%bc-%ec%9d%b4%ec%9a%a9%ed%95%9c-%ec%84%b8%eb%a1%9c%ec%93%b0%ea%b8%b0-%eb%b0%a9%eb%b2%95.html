1. CSS3에서 제공하는 세로쓰기 방법

CSS3의 writing-mode는 다국어를 위한 다양한 쓰기 방법을 지원한다.

CSS Writing Modes Level 3 defines CSS features to support various international script modes, such as left-to-right (e.g., Latin and Indic), right-to-left (e.g., Hebrew and Arabic), bidirectional (e.g., mixed Latin and Arabic) and vertical (e.g., Asian). This article is about the CSS writing-mode property.
출처: https://developer.mozilla.org/en-US/docs/Web/CSS/writing-mode

위 출처의 글을 읽어보면 writing-mode: vertical-rl;  라는 CSS 속성을 이용하면 세로쓰기를 할 수 있다고 하는데, 막상 해보니 안된다. 벤더 프리픽스를 붙여주니 작동한다.

2. 세로쓰기 예

코드

<!doctype html>
<html>
<head>
	<meta http-equiv="Content-Type" content="text/html; charset=utf-8" />
	<link rel="stylesheet" type="text/css" href="styles.css" />
</head>
<body>
<div class="kor">나는 우리나라가 세계에서 가장 아름다운 나라가 되기를 원한다. 가장 부강한 나라가 되기를 원하는 것은 아니다. 내가 남의 침략에 가슴이 아팠으니, 내 나라가 남을 침략하는 것을 원치 아니한다.</div>
</body>
</html>
.kor {
	width: 400px;
	height: 250px;
	border: 1px solid #aaa;
	font-family: "함초롬바탕 LVT";
	font-size: 16pt;

	-webkit-writing-mode: vertical-rl;
	-moz-writing-mode: vertical-rl;
	-ms-writing-mode: tb-rl;
	-ms-writing-mode: vertical-rl;
	writing-mode: vertical-rl;
}

tb-rl은 이제 writing-mode 값으로 쓰이지 않는다. vertical-rl을 사용하면 된다.

결과

실제로는 writing-mode은 아직 동작하지 않고, 벤더 프리픽스를 붙여주어야만 된다.

사파리와 크롬, 오페라에서는 동작하고, 파이어폭스는 지원하지 않는다(nightly 버전도 안됨). IE는 확인 안함.

덧

사실 처음 예제로 준비한 파일은 옛한글이 들어간 글이었는데, 구글 크롬에서 실행하니 맥이 엄청 느려졌다. 아마 옛한글을 보여주기 위해 적절한 폰트를 찾는게 아닌가 싶다. 이 문제는 나중에 살펴봐야겠다. 오늘은 눈이 빠져서 더 못해먹겠다.
