AWS EC2에서 xelatex으로 텍 파일을 컴파일하는데, 마지막 PDF 생성할 때 "xelatex: I can't write on file `test.pdf'." 오류가 발생한다.

latex - dvipdfmx는 정상, latex - xdvipdfmx도 정상, pdflatex도 정상이다. xelatex만 저 오류가 발생한다.

메모리 문제일 거라고는 생각하지 못했는데 strace xelatex test 명령을 실행하고 살펴보니

Cannot allocate memory

오류가 발생한다.

구글링해보니 스왑을 만들어서 하라고... 다음 두 글을 참조해서 해결할 수 있었다.

  • xelatex: I can't write on file `test.pdf'.
  • Amazon EC2 and MongoDB configuration for great performance
